# 《智能程序设计-C》课程设计项目文档

## 项目名称：极简三国杀·人格分裂AI

---

## 一、项目概述

本项目实现一款基于C语言的**简化版三国杀**游戏，核心特色是**AI具有多个人格**（激进派、保守派、赌徒派），每局游戏AI会随机展现一种人格，并通过**动态心理活动文字**解释其决策逻辑。游戏支持两种模式（单挑、非对称主公局），包含三名武将（张飞、赵云、诸葛亮）及基本卡牌（杀、闪、桃、过河拆桥、无中生有），满足课程设计对文件操作、数据操作、智能策略的全部要求。

---

## 二、游戏模式

|模式|说明|玩家可选角色|
|---|---|---|
|**单挑模式**|玩家 vs 1个AI，公平对战（无主公增益）|玩家从三名武将中选一个，AI随机选一个（人格随机）|
|**非对称模式**|主公 vs 2个反贼。主公获得增益：摸牌阶段摸3张（原2张），血量上限+1|玩家可选择当**主公**（对抗2个AI反贼）或当**反贼**（与另一个AI反贼共同对抗AI主公）|

## 三、武将及技能

| 武将      | 体力上限 | 技能                                                                                        |
| ------- | ---- | ----------------------------------------------------------------------------------------- |
| **张飞**  | 4    | **咆哮**：出牌阶段，你使用的【杀】无次数限制。                                                                 |
| **赵云**  | 4    | **龙胆**：你可以将【杀】当【闪】、【闪】当【杀】使用或打出。                                                          |
| **诸葛亮** | 3    | **观星**：准备阶段，你可以观看牌堆顶的X张牌（X为存活角色数），将这些牌以任意顺序置于牌堆顶或牌堆底。  <br>**空城**：锁定技，若你没有手牌，你不能成为【杀】的目标。 |

> 玩家和AI均从这三名武将中选择，AI的人格独立于武将。

## 四、卡牌及牌堆

共30张牌，初始牌堆从文件`cards.txt`读取，支持自定义。

|牌名|数量|效果|
|---|---|---|
|杀|12|对一名其他角色使用，目标需出闪，否则受到1点伤害|
|闪|8|抵消杀|
|桃|5|出牌阶段使用，回复1点体力（不可超过体力上限）；濒死时可用于救回角色|
|过河拆桥|3|弃置目标角色一张手牌|
|无中生有|2|摸两张牌|

## 五、回合流程

每回合按顺序执行以下阶段：

1. **准备阶段**：触发“观星”等技能。
    
2. **摸牌阶段**：摸2张牌（若为主公则摸3张）。
    
3. **出牌阶段**：
    
    - 可任意使用锦囊牌（无中生有、过河拆桥）。
        
    - 使用【杀】的次数限制：普通角色一回合只能使用1次；张飞无限制。
        
    - 可使用【桃】回复体力（不能超过上限）。
        
4. **弃牌阶段**：手牌数大于当前体力值时，弃至等于体力值。
    
5. **结束阶段**：无额外效果。

## 六、AI人格分裂设计

AI拥有三种人格，每局游戏开局随机选择一种，并通过**动态心理活动文字**体现。

|人格|策略倾向|决策权重示例|心理活动风格|
|---|---|---|---|
|**激进派**|优先进攻，敢于冒险|杀权重×1.5，桃仅在血量≤2时使用|“杀！进攻是最好的防守！”|
|**保守派**|优先防守，求稳|闪权重提高，桃在血量≤3时使用，杀仅在对手手牌少时使用|“稳妥一点，先留个闪。”|
|**赌徒派**|随机决策，喜欢赌博|随机扰动±20%分数，有时故意不出牌|“赌一把！无中生有！”|

> 实现方式：待定！！
> 例如：为每张牌计算基础分数，再乘以人格权重系数或添加随机扰动，最终选择最高分牌。不采用强制规则，避免AI显得愚蠢。

## 七、动态心理活动
实现方式待定！！

AI每次出牌（或决定不出牌）时，会在界面显示一句根据**当前人格、所出牌型、双方血量/手牌数**动态生成的文字。  
示例：

- 激进派出杀且对手血量≤2：_“你只剩2血了，这一刀你躲不开！”_
    
- 保守派吃桃且自己血量≤2：_“血量危急，赶紧吃个桃回血。”_
    
- 赌徒派出过河拆桥：_“赌你手里没闪！”_
    

实现方式：为每种（人格+牌型+条件）准备模板，填充变量后输出。

## 八、胜负与濒死求桃

### 濒死流程

当一名角色体力值降至0或以下时，进入濒死状态：

- 从当前回合角色开始，按逆时针顺序，每个角色可依次使用【桃】救回（每桃回复1点）。
    
- 若体力值恢复至≥1，则脱离濒死；否则角色死亡。
    

### 胜负判定

- **主公死亡** → 反贼胜利（游戏立即结束）。
    
- **所有反贼死亡** → 主公胜利（游戏立即结束）。
    
- 单挑模式下，一方死亡即另一方胜利。

## 十、智能特性体现

1. **人格分裂**：同一武将的AI在不同人格下表现出截然不同的策略倾向。
    
2. **动态心理活动**：AI解释自己的决策依据，增强可解释性。
    
3. **加权评分决策**：基于多因素的智能选择，而非简单随机。
    
4. **可选扩展**：后续可加入“学习机制”，根据玩家历史胜率选择克制人格。
